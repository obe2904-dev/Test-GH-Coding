# v4.7.3 - tone_model DB Constraint Fix

## Problem Statement

The brand-profile-generator Edge Function was failing with **500 errors** when attempting to save invalid `tone_model` structures to the database:

```
Failed to save brand profile: new row for relation "business_brand_profile" 
violates check constraint "tone_model_valid_structure_v2"
```

This happened because:
1. AI sometimes generated invalid tone_model structures (wrong types, missing fields, arrays too short)
2. The inline normalization in `parseBrandProfileResponse()` was insufficient
3. When DB rejected the insert, the function crashed with 500 (bad UX)

## Solution Implemented

### 1. Created `tone-model.ts` Sanitizer Module

**Location**: `supabase/functions/_shared/brand-profile/tone-model.ts`

**Key Functions**:
- `sanitizeToneModelForDb(input: any, languageCode: string): ToneModelV2 | null`
  - Takes potentially invalid input
  - Returns fully normalized object OR null
  - Never returns invalid structures
  - Validates all DB constraints:
    - `primary_keywords`: 2-6 strings
    - `writing_rules`: 3-8 strings
    - `good_examples`: 2-6 strings
    - `avoid_examples`: 2-6 strings
    - `formality`: 'formal' | 'informal' | 'mixed'
    - `emoji_level`: 'none' | 'minimal' | 'moderate' | 'frequent'
    - `source`: 'website' | 'manual' | 'hybrid'
    - `confidence`: 'high' | 'medium' | 'low'
    - `version`: Always "2.0"
    - `language`: Short code (e.g., 'da', 'en')
    - `generated_at`: Current timestamp (ISO 8601)
    - `notes`: Optional, max 500 chars

- `runToneModelSanitizerTests()`
  - Runtime tests executed on function startup
  - Validates sanitizer logic with 8 test cases
  - Ensures correctness before handling requests

#### c) Added Error Handling with Retry
**New try/catch around save**:
```typescript
try {
  await saveBrandProfile(/* ... */)
} catch (saveError: any) {
  const errorMsg = String(saveError?.message || '')
  const isToneModelConstraint = errorMsg.includes('tone_model_valid_structure_v2')
  
  if (isToneModelConstraint) {
    console.error(`[${requestId}] ❌ DB rejected tone_model structure:`, errorMsg)
    
    // Retry with tone_model as null
    console.log(`[${requestId}] 🔄 Retrying save with tone_model=null...`)
    brandProfile.tone_model = null
    
    await saveBrandProfile(/* ... */)
    
    requestErrors.add(
      ErrorCategory.VALIDATION_FAILED,
      ErrorSeverity.HIGH,
      'tone_model failed DB constraint; saved as null',
      'save',
      { originalError: errorMsg }
    )
  } else {
    throw saveError
  }
}
```

## Test Cases Covered

The sanitizer includes 8 runtime tests:

1. ✅ **null input** → returns null
2. ✅ **undefined input** → returns null
3. ✅ **empty object** → returns null
4. ✅ **arrays with wrong types** (numbers instead of strings) → returns null
5. ✅ **missing required fields** → returns null
6. ✅ **arrays too short** (< minimum length) → returns null
7. ✅ **correct object** → returns normalized ToneModelV2
8. ✅ **invalid enums** → normalizes to defaults

## Behavior Guarantees

### ✅ Function Will NEVER:
- Return 500 due to tone_model shape issues
- Crash during save if tone_model is invalid
- Fabricate invalid structures to satisfy DB constraints
- Double-wrap JSON or corrupt JSONB columns

### ✅ Function Will ALWAYS:
- Return 422 for validation issues (not 500)
- Log detailed diagnostic information
- Retry save with `tone_model=null` if constraint fails
- Either save valid tone_model OR save null (both allowed by DB)
