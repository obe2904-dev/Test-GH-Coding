## Critical Bugs to Fix

### 1. Percentage Display Bug ❌ CRITICAL
**Location:** `src/components/brandProfile/ProgrammeCard.tsx`

**Problem:** Content affinity shows 3000%, 4000% instead of 30%, 40%

**Root Cause:** 
- Database stores values as 0-100 (30, 40, 70)
- Display code multiplies by 100: `(value * 100)%`
- Result: 30 * 100 = 3000%

**Fix:** Remove multiplication - values are already percentages
```typescript
// WRONG (current):
<span>{((value as number) * 100).toFixed(0)}%</span>

// CORRECT:
<span>{(value as number).toFixed(0)}%</span>
```

**Also affects:** 
- Progress bar widths (currently OK because they cap at 100%)
- But should be consistent

### 2. Missing Layer 3 Fields ⚠️ HIGH PRIORITY
**Location:** `src/pages/dashboard/BrandProfilePageV5.tsx` or `src/components/brandProfile/BrandProfileDisplay.tsx`

**Problem:** V5 generates 6 Layer 3 fields, but only 1-2 are displayed

**Generated by V5 (from identity-profile.ts):**
```typescript
{
  brand_essence: string;           // ✅ SHOWN (at top, unlabeled)
  positioning: string;              // ✅ SHOWN (in collapsible section)
  core_values: string[];            // ❌ MISSING

  identity_confidence: number;      // ❌ MISSING (0.85 = 85%)
  identity_reasoning: string;       // ❌ MISSING
}
```

**Fix:** Add section to display all Layer 3 fields
