## 🎯 YOUR QUESTIONS ANSWERED

### Question 1: When does Geographic Context get populated?

**ANSWER: It's HARDCODED, not AI-generated**

The geographic context you see is **pre-written by developers**, not generated by AI.

**Where it's defined:**
`supabase/functions/_shared/brand-profile/geographic-context.ts`

**What happens:**
1. **Developer writes** city profiles (one-time, manually)
2. **System looks up** city from postal code
3. **System assembles** narrative from pre-written parts
4. **System stores** in Layer 0 intelligence

---

#### Geographic Context Flow

```
STEP 1: POSTAL CODE LOOKUP (Logic)
─────────────────────────────────
Input:  8000 (postal code)
Logic:  if (code >= 8000 && code <= 8270) return 'Aarhus'
Output: "Aarhus"


STEP 2: CITY PROFILE LOOKUP (Hardcoded Data)
──────────────────────────────────────────────
Input:  "Aarhus"
Lookup: DANISH_CITY_PROFILES['Aarhus']
Output: {
  city: 'Aarhus',
  population: 350000,
  size_category: 'medium_city',
  characteristics: ['university_town', 'second_city', ...],
  tone_guidance: 'Casual og tilgængelig. Balance mellem urban cool...',  // ← HARDCODED
  cultural_context: 'Danmarks næststørste by, stor studiepopulation...',  // ← HARDCODED
  competition_level: 'high'
}


STEP 3: LOCATION INFERENCE (Pattern Matching)
──────────────────────────────────────────────
Input:  "ved åen"
Logic:  if (ref.match(/ved åen|ved havnen|ved stranden/))
Output: {
  type: 'waterfront_leisure',
  signature: 'ved åen',
  advantages: ['scenic location', 'outdoor seating', ...],
  tone_implications: 'Fremhæv location kraftigt - det er jeres USP...'  // ← HARDCODED
}


STEP 4: NARRATIVE ASSEMBLY (String Concatenation)
──────────────────────────────────────────────────
Input:  City profile + Location context
Logic:  Concatenate pre-written strings
Output: "GEOGRAFISK CONTEXT:\n\nBy: Aarhus (medium_city, 350.000 indbyggere)..."
        ↑ NO AI INVOLVED - just string assembly
```

---

#### What's Hardcoded vs What's AI-Generated

**HARDCODED (Written by developer):**
✅ City profiles (København, Aarhus, Odense, Aalborg, Varde)
✅ Tone guidance per city
✅ Cultural context descriptions
✅ Location type patterns (waterfront, downtown, etc.)
✅ Tone implications per location type
✅ Professional persona base prompts ("Du er en...")
✅ Voice archetype rules

**AI-GENERATED (Generated per business):**
❌ NONE in geographic context!
✅ Brand profile content (menu descriptions, atmosphere, etc.)
✅ Commercial orientation per programme
✅ Audience profiles

**File:** `geographic-context.ts` Line 118


**When Brand Profile V5 is generated:**
1. System reads postal code "8000"
2. System looks up → finds "Aarhus"
3. System retrieves this HARDCODED profile
4. System assembles narrative from these PRE-WRITTEN strings
5. System stores in database (Layer 0)

**NO AI INVOLVED in this step!**

---

#### Why Hardcoded?

**Advantages:**
✅ Consistent quality - you control the tone guidance
✅ Fast - no AI call needed
✅ Predictable - same city always gets same guidance
✅ Cost-effective - no tokens spent
✅ Maintainable - update in one place

**The Alternative (AI-generated) would be:**
❌ Expensive - AI call for every business
❌ Inconsistent - different guidance for same city
❌ Unpredictable - quality varies
❌ Slower - extra API calls

### Question 2: How to separate Persona vs What Persona Does?

**ANSWER: Three-layer structure**

```
LAYER 1: WHO YOU ARE (Core Persona)
────────────────────────────────────
"Du er en professionel social media manager specialiseret i [domain]."

Purpose: Establishes identity and expertise
Stays same across: Brand Profile, Ideas, Texts
Location: professional-persona.ts → system_persona


LAYER 2: WHAT YOU KNOW (Context)
─────────────────────────────────
Business facts + Geographic context + Voice rules

Purpose: Provides specific knowledge about THIS business
Stays same across: All tasks for this business
Location: Layer 0 Intelligence (database)


LAYER 3: WHAT YOU'RE DOING NOW (Task)
──────────────────────────────────────
"You are generating a brand profile" OR
"You are writing a social media post about X" OR
"You are suggesting quick ideas"

Purpose: Defines current task and output format
Changes per: Each specific function call
Location: Function-specific prompt builders
```

## 🔄 HOW TO USE ACROSS FUNCTIONS

### Pattern: Same Persona + Different Tasks

```typescript
// SHARED BASE (always same for this business)
const corePersona = layer0.professional_persona.system_prompt
const voiceRules = layer0.voice_archetype.base_rules
const geoContext = layer0.geographic_context.narrative

// FUNCTION-SPECIFIC TASK
const brandProfileTask = `
CURRENT TASK: Generate comprehensive brand profile
OUTPUT: Structured JSON with menu analysis, atmosphere, etc.
`

const textGenerationTask = `
CURRENT TASK: Write social media post about ${contentType}
OUTPUT: 180-300 character caption with emoji
LENGTH: ${lengthRequirement}
`

const quickSuggestionsTask = `
CURRENT TASK: Generate 3 quick post ideas for this week
OUTPUT: JSON array with {title, contentType, reasoning}
`

// ASSEMBLE FULL PROMPT
const fullPrompt = `
${corePersona}

${geoContext}

VOICE RULES:
${voiceRules.map(rule => `- ${rule}`).join('\n')}

${taskSpecificInstructions}
`
```

## 💡 KEY INSIGHTS

### 1. Geographic Context is Pre-Written, Not AI

**Facts are hardcoded:**
- City populations
- City characteristics
- Tone guidance per city
- Cultural context descriptions
- Location type patterns

**Benefits:**
- Consistent quality
- No AI costs
- Fast lookup
- Easy to maintain
- You control the narrative

**To Update:**
Edit `geographic-context.ts` → Redeploy → New businesses get updated guidance

---

### 2. Persona Should Be Portable

**The "Du er en..." prompt should work for:**
- ✅ Brand Profile Generation (currently works)
- ✅ Text Generation (needs integration)
- ✅ Quick Suggestions (needs integration)
- ✅ Any future AI function

**How:**
Store once in Layer 0 → Load in all functions

**Current Reality:**
- Brand Profile: Uses Layer 0 ✅
- Text Generation: Uses generic prompt ❌
- Quick Suggestions: Uses generic prompt ❌

---

### 3. Three-Layer Prompt Pattern

**LAYER 1: Identity** (who you are)
- Professional domain
- Years of experience
- Core expertise

**LAYER 2: Knowledge** (what you know)
- This specific business
- Geographic context
- Voice rules to follow

**LAYER 3: Task** (what you're doing now)
- Generate profile / Write post / Suggest ideas
- Output format requirements
- User input/request

**ALL THREE combine to make the full prompt**

## ✅ SUMMARY

### Question 1: When does geographic context get populated?

**Answer:**
- Geographic context is **HARDCODED** by developers
- NOT generated by AI
- Tone guidance is **pre-written** in `geographic-context.ts`
- System **looks up and assembles** from pre-written parts
- Populated during **Brand Profile V5 generation**
- Stored in **Layer 0 intelligence** in database

---

### Question 2: How to separate persona vs what persona does?

**Answer:**
- **Persona** = "Du er en..." (WHO - stays same)
- **Context** = Business facts + voice rules (WHAT YOU KNOW - stays same per business)
- **Task** = Current action (WHAT YOU'RE DOING - changes per function)

**Solid Structure:**
```
PART 1: Core professional persona (Layer 0 system_prompt)
PART 2: Business context + voice rules (Layer 0 data)
PART 3: Task-specific instructions (function-specific)
```

**Usage:**
- Same persona across Brand Profile, Ideas, Texts
- Load from Layer 0 once
- Add task-specific instructions per function
- Total prompt = Persona + Context + Task
