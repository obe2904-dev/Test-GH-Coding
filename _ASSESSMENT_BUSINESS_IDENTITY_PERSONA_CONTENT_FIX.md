## Problem Analysis

### Issue 1: "Bæredygtighed" Without Evidence 🚨

**Where it appears**:
- `identity_keywords` field (3 identity chips generated by AI)
- Potentially in persona text if AI infers from keywords

**Why it's wrong**:
According to [v5-prompts.ts](supabase/functions/_shared/brand-profile/v5-prompts.ts#L168-L169):
```typescript
// RULE 5: VALUES MUST HAVE EVIDENCE
❌ "Bæredygtighed" (no evidence)
✅ "Bæredygtighed" (if menu shows local suppliers, organic, zero-waste)
```

**Problem**: AI generates "Bæredygtighed" as an identity keyword **without verifying** menu evidence:
- No "lokale råvarer" mentioned in menu items
- No "økologisk" certifications visible
- No "zero-waste" or sustainability claims in Om Os text
- **Result**: Hallucinated value that doesn't match business reality

**Impact**:
- ❌ Text generation gets incorrect identity context
- ❌ Posts may reference sustainability when venue doesn't focus on it
- ❌ Brand voice misalignment (claiming values not backed by operations)

### The Mismatch Problem ⚠️

| Aspect | `strategic_audience_segments` | `business_identity_persona` | Issue |
|--------|------------------------------|----------------------------|-------|
| **Primary segment name** | "Aftensmad ved åen" | "Frokost-pendlere" | ❌ DIFFERENT |
| **Primary timing** | "Weekday 18-22" | "Hverdage 12:00-14:00" | ❌ DIFFERENT |

**Root Cause Analysis**:

1. **Outdated persona regeneration**: Persona was generated BEFORE `strategic_audience_segments` was populated
2. **AI rewording**: AI paraphrases segment names instead of using exact labels from input
3. **No validation**: No check that persona text matches structured JSONB data

**Impact**:
- ❌ **Inconsistent targeting**: Prompt says "Frokost-pendlere" but logic uses "Aftensmad ved åen"
- ❌ **Confused AI context**: Text generation gets conflicting audience signals
- ❌ **Data integrity**: Same business described differently in same table

## Proposed Solution

### Fix 1: Remove "Bæredygtighed" Hallucination

**Detection Strategy**:
```sql
-- Find businesses with "bæredygtighed" in identity_keywords but NO evidence
SELECT 
  bbp.business_id,
  bbp.identity_keywords,
  b.name,
  -- Check for evidence in various fields
  bbp.business_identity_persona LIKE '%lokal%råvar%' as has_local_evidence,
  bbp.business_identity_persona LIKE '%økolog%' as has_organic_evidence,
  bbp.business_identity_persona LIKE '%bæredygt%' as has_sustainability_mention,
  'bæredygtighed' = ANY(bbp.identity_keywords) as has_keyword
FROM business_brand_profile bbp
JOIN businesses b ON b.id = bbp.business_id
WHERE 'bæredygtighed' = ANY(bbp.identity_keywords);
```

**Remediation Options**:

#### Option A: **Manual Verification** (Recommended for small dataset)
1. Review each business with "bæredygtighed" keyword
2. Check menu items, Om Os text, website for evidence
3. If NO evidence → remove from `identity_keywords`
4. Regenerate persona with corrected keywords

#### Option B: **AI Re-validation** (Scalable for large dataset)
1. Create validation function that checks:
   - Menu items for "lokal", "økologisk", "bæredygtig" keywords
   - Om Os text for sustainability claims
   - Features flags (if any)
2. Auto-remove "bæredygtighed" if no evidence found
3. Regenerate personas in batch

#### Option C: **Strengthen Generation Prompt** (Preventive)
Add to identity_keywords generation rules in [prompt-b.ts](supabase/functions/_shared/brand-profile/prompts/prompt-b.ts#L247):
```typescript
IDENTITY KEYWORDS (identity_keywords):
- Exactly 3 words. Each must occupy a DIFFERENT dimension:
  1. Atmosphere dimension (hygge, intim, levende, rolig, urban...)
  2. Formality dimension (uformel, casual, lavmælt, professionel...)
  3. Category/format dimension (klassikere, specialty, håndværk, fusion...)
- ANTI-PATTERN: all synonyms e.g. "Hygge · Samvær · Fællesskab" = WRONG
- Must NOT overlap with tone_model.primary_keywords (those describe writing STYLE, not identity)
+ EVIDENCE REQUIREMENT: Do NOT include value-based keywords (bæredygtighed, kvalitet, fællesskab) 
+   UNLESS you have explicit evidence in menu items, Om Os text, or website content
+ ❌ "Bæredygtighed" (if no local/organic/sustainability claims visible)
+ ✅ "Bæredygtighed" (if menu shows "lokale råvarer", "økologisk", or zero-waste practices)
```

**Recommendation**: **Combination approach**:
1. ✅ Strengthen prompt (Option C) — prevents future hallucinations
2. ✅ Manual verification for Café Faust — immediate fix
3. 🔜 AI re-validation script — clean up existing data at scale

**Current Mismatch Detection**:
```sql

  strategic_audience_segments->'primary'->>'name' as jsonb_primary_name,

  business_identity_persona LIKE '%' || (strategic_audience_segments->'primary'->>'name') || '%' as persona_has_primary,
```
