# Formality Level Fix Assessment

**Date:** June 12, 2026  
**Business:** Café Faust (f4679fa9-3120-4a59-9506-d059b010c34a)  
**Issue:** Inconsistent formality indicators  
**Fix:** Updated `voice.formality_level` from "semi-formal" to "informal"

---

## Problem Identified

### Inconsistency in Formality Signals

The brand profile had conflicting formality indicators:

```
❌ BEFORE:
  voice.formality_level: "semi-formal"                           (inconsistent)
  voice.tone_dna.owner_voice.register_level: "casual"            (correct)
  voice.tone_dna.culinary_character.formality_requirement: "casual" (correct)
  voice.tone_dna.recommended_tone.tone_positioning: "Casual-varm..." (correct)
```

**3 out of 4 indicators** pointed to "casual/informal", but the main `formality_level` field was "semi-formal".

---

## Root Cause Analysis

### Why Did This Happen?

1. **AI Generation Inconsistency**
   - The `formality_level` is generated by GPT-4o based on business context
   - The AI prompt specifies: `"formality_level": "informal" | "semi-formal" | "formal"`
   - The tone_dna is generated by a different prompt with more detailed analysis
   - Result: Two different AI calls made different formality assessments

2. **Context Analysis**
   - **Café Faust profile:**
     - Hybrid café/restaurant (all-day dining)
     - Waterfront location (95 score)
     - Mid-range pricing (moderate)
     - Student + tourist demographics
     - Casual dining character
   - **Expected formality:** Informal (matches casual positioning)
   - **AI assessment:** Semi-formal (AI likely saw "bistro" or "European fusion" and added formality)

3. **Prompt Guidance Gap**
   - The voice prompt doesn't explicitly cross-check against tone_dna analysis
   - No validation to ensure `formality_level` aligns with `tone_dna.culinary_character.formality_requirement`

---

## Solution Implemented

### Direct Database Update ✅

```javascript
// Changed:
v5.voice.formality_level = 'informal'
```

### Result:

```
✅ AFTER:
  voice.formality_level: "informal"                              (aligned)
  voice.tone_dna.owner_voice.register_level: "casual"            (aligned)
  voice.tone_dna.culinary_character.formality_requirement: "casual" (aligned)
  voice.tone_dna.recommended_tone.tone_positioning: "Casual-varm..." (aligned)
```

**All 4 formality indicators now point to "casual/informal"** ✅

The V5 brand profile uses these formality_level values:
- `"informal"` ← Corresponds to "casual"
- `"semi-formal"` ← Professional but approachable
- `"formal"` ← Distance, De-form

### Internal Aliases

## Long-Term Prevention

### Recommended System Improvements

1. **Cross-Validation Rule**
   - After generating `formality_level`, compare against `tone_dna.culinary_character.formality_requirement`
   - If mismatch, use tone_dna value (it has more detailed context)

2. **Enhanced Prompt Context**
   - Include tone_dna formality requirement in the voice generation prompt
   - Example: "The detailed culinary analysis determined formality should be 'casual' - ensure formality_level aligns"

3. **Post-Generation Validation**
   - Add check: `if (voice.formality_level !== mapFormalityFromToneDNA(tone_dna)) { ... }`
   - Log warning when mismatch detected

4. **Prompt Strengthening**
   - Add explicit guidance for hybrid cafés: "All-day dining with casual positioning → informal"
   - Add examples showing business type → formality mapping
