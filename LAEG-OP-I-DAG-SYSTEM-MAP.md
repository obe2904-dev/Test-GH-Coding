# "LÆG OP I DAG" - SYSTEM ANALYSIS

## OVERVIEW
The "Læg op i dag" system consists of two phases:
1. **IDEA GENERATION** (`get-quick-suggestions`) - Creates 3 daily post ideas
2. **TEXT GENERATION** (`generate-text-from-idea`) - Converts selected idea into caption text

---

## PHASE 1: IDEA GENERATION (get-quick-suggestions)

### DATABASE SOURCES

**Core Business Data:**
- `businesses` table:
  - `name` - Business name
  - `vertical` - Business type (cafe, restaurant, etc.)
  - `plan` - Subscription tier (free/standardplus/premium)

**Location & Operations:**
- `business_locations` table:
  - `city` - City name
  - `country` - Country (for language detection)
  
- `business_operations` table:
  - `opening_hours_json` - Daily opening/closing times
  - `kitchen_close_time` - When kitchen stops serving
  - `price_level` - 1-4 scale (budget to premium)
  - `reservation_required` - Boolean
  - `accepts_walk_ins` - Boolean

**Menu Data (Paid tier only):**
- `menu_results_v2` table:
  - `structured_data` - JSONB array of menu sections
  - Each menu item contains:
    - `item_name` - Dish name
    - `item_description` - Ingredients/preparation
    - `category` - Menu category
    - `service_period` - When served (brunch/frokost/middag)

**Brand Profile (Paid tier only):**
- `business_brand_profile` table:
  - `brand_profile_v5` - JSONB containing:
    - `voice.personality_traits` - Tone descriptors
    - `voice.structural_rules` - Writing constraints
    - `voice.style_rules` - Style guidelines
    - `writing_examples.signature_phrases` - Brand-specific phrases
    - `identity.business_description` - Business character
    - `identity.category_keywords` - Identity keywords
    
- `business_audience_segments` table:
  - Day/hour timing patterns
  - Segment motivation (why they visit)
  - Content angles for each segment

**Location Intelligence (Paid tier only):**
- `location_intelligence` table:
  - `category_scores` - Proximity scores (waterfront, tourist, etc.)
  - `density_text` - Nearby hospitality competition
  - `tourist_factor` - International visitor likelihood

**External APIs:**
- OpenWeather API:
  - Current conditions
  - 5-hour forecast
  - Temperature, description

### FREE TIER vs PAID TIER

**FREE TIER LIMITATIONS:**
- ❌ No menu data (only sees dish names from title)
- ❌ No brand profile (uses safe fallback tone)
- ❌ No opening hours (estimates typical hours)
- ❌ No location intelligence
- ❌ No audience segments
- ✅ Weather data
- ✅ Basic business info (name, city, vertical)

**PAID TIER GETS:**
- ✅ Full menu with descriptions
- ✅ Complete brand voice profile
- ✅ Actual opening hours
- ✅ Location context
- ✅ Audience segment timing
- ✅ Booking pattern analysis
- ✅ Weather data
- ✅ Business operations data

### AI PROMPT STRUCTURE (IDEA GENERATION)

**System Instruction:**
- Role: "Du er social media-rådgiver"
- Task: Generate 3 post ideas (slots A, B, C)
- Format: JSON with specific fields

**Slot Assignments:**
- **Slot A (offering)**: Menu item or seasonal offering
- **Slot B (guest_moment)**: Social occasion or timing
- **Slot C (behind_scenes)**: Team, service, or preparation

**Data Injected into Prompt:**

1. **BUSINESS CONTEXT** (all tiers):
   - Business name, city, vertical
   - Current date, time, weather

2. **MENU SECTION** (paid only):
   - Top 5 menu items with descriptions
   - Service periods (brunch/frokost/middag)
   - Free tier: "No menu data available"

3. **VOICE GUIDANCE** (paid only):
   - Personality traits
   - Signature phrases (if not about specific dishes)
   - Free tier: Generic friendly tone

4. **TIMING CONTEXT** (paid only):
   - Opening hours
   - Active audience segment
   - Segment motivation
   - Free tier: Generic "daytime cafe" assumptions

5. **SLOT-SPECIFIC RULES**:
   - Free tier Slot C (behind_scenes):
     ```
     DU ER OBSERVATØR — ikke historiefortæller.
     GRÆNSE: Du må KUN rapportere verificerbare facts.
     KONSEKVENS: Hvis du ikke kan verificere det i data, eksisterer det ikke.
     
     Eksempel:
     - Data viser: "BURGER med hakkebøf"
     - Du skriver: "Børnemenu med burger og fritter" ✅
     - Du skriver IKKE: "Hænder former burgerboller" ❌ (opfundet)
     ```

   - Paid tier Slot C:
     ```
     VIS MENNESKENE bag oplevelsen — teamet, service, relation til gæster.
     Du må frit beskrive processer og følelser.
     ```

### OUTPUT SAVED TO DATABASE

**Stored in `daily_suggestions` table:**
- `title` - Post headline (generated by AI)
- `menu_item_name` - Dish name (if menu post)
- `menu_item_description` - Ingredients (from DB, NOT AI invention)
- `content_type` - menu_item, behind_scenes, occasion, etc.
- `caption_base` - Ingredient brief or concrete anchor
- `why_explanation` - Why this works now
- `photo_idea` - Photography suggestion
- `suggested_time` - When to post (HH:MM)
- `cta_intent` - visit/social/engagement
- `weather_forecast` - Weather at generation time

**Key Safeguard:**
```typescript
// DB description takes priority over Gemini's dish_text_brief
// Gemini invents ingredients from training data
const trustedDesc = menuDescriptionMap.get(menuItemName)
dishTextBriefs[idx] = trustedDesc || geminiInvention || ''
```

## PHASE 2: TEXT GENERATION (generate-text-from-idea)

### DATABASE SOURCES

**Same business tables as Phase 1, PLUS:**

**Post Drafts:**
- `daily_suggestions` table row (selected by user):
  - `title` - The headline
  - `menu_item_name` - Dish name
  - `menu_item_description` - Trusted description from DB
  - `caption_base` - Ingredient brief or anchor
  - `content_type` - Post category
  - `why_explanation` - Strategic rationale
  - `weather_forecast` - Weather at idea generation

**Additional Context Resolution:**
- Menu lookup fallback (if description missing)
- Token extraction from title (emergency dish name recovery)
- Signature phrase filtering (removes phrases about wrong dishes)

### FREE TIER vs PAID TIER

**FREE TIER PROMPT:**
```
DU ER OBSERVATØR — ikke historiefortæller.

ROLLE: Du beskriver præcist hvad der ER verificerbart.
GRÆNSE: Du har KUN rettens navn og brugerens beskrivelse. 
        Du har INGEN detaljeret menudata.
KONSEKVENS: Hvis du ikke kan verificere det i data, eksisterer det ikke.

REGLER:
1. Skriv KUN hvad du kan verificere:
   - Rettens navn ✅
   - Detaljer fra brugerens titel ✅
   - At retten serveres her ✅
2. OPFIND ALDRIG:
   - Ingredienser der ikke står i titel ❌
   - Tilberedningsmetoder ❌
   - Brødtyper, sovse, garniturer ❌
   - Tekstur-beskrivelser ("sprød", "saftig") ❌

EKSEMPEL:
- Titel: "Faustburger med sprød bacon"
- Du skriver: "Faustburger med bacon serveres" ✅
- Du skriver IKKE: "briochebolle", "chilimayo", "syltede rødløg" ❌
```

**Model:** GPT-4o-mini
**Temperature:** 0.0 (deterministic, no creativity)
**Length:** 150-280 characters

**PAID TIER PROMPT:**
```
Du skriver som [BusinessName]s stemme — indefra, ikke udefra.
Du er ekspert i at skrive autentiske social media-tekster 
der afspejler stedet uden at sælge på det.

TONE: [personality traits fra brand profile]
WRITING RULES: [structural_rules fra V5]
GOOD EXAMPLES: [good_examples fra V5]
CONTENT ANCHORS: [faktuelle holdepunkter]
EMOJI: [emoji_level guidance]
```

**Model:** GPT-4o
**Temperature:** 0.7 (creative variation)
**Length:** Varies by content type

### DATA FLOW IN PROMPT

**1. SYSTEM INSTRUCTION:**
- Free: OBSERVER persona (no invention)
- Paid: Brand voice + creative freedom

**2. BUSINESS CONTEXT:**
- Business name, city, vertical
- Opening hours (paid only)

**3. HOOK:**
- For menu posts: Dish name + title
- For BTS posts: Title + concrete anchor
- Free: Limited to title only

**4. CONTENT BLOCK (INDHOLD):**
- Menu description (paid only, from DB)
- Free: Empty or "No menu data available"

**5. VOICE GUIDANCE (paid only):**
- Writing rules (5 max)
- Good examples (3 max)
- Avoid examples (3 max)
- Signature phrases (filtered to context)
- Emoji instruction

**6. LENGTH REQUIREMENT:**
- Menu posts: 180-350 chars
- BTS posts: 150-280 chars
- Atmosphere: 150-280 chars

**7. CTA:**
- Booking-required: "Book bord"
- Walk-in friendly: "Kom forbi"
- Social: "Tag en ven med"
- None if atmosphere/BTS

### CRITICAL SAFEGUARDS

**Menu Description Sanitization:**
```typescript
// Strip upsell references so AI can't "see" other dishes
// Remove marketing summary sentences
sanitizeMenuDesc(rawDescription)
```

**Signature Phrase Filtering:**
```typescript
// Only include phrases relevant to current dish
// Prevents AI from writing about wrong menu items
filterSignaturePhrasesToContext(allPhrases, menuItemName)
```

**First Sentence Extraction:**
```typescript
// Extract concrete opening from caption_base
// Skip all-caps headlines
// Prioritize sentences with normal case ratio
extractFirstSentence(captionBase)
```

### OUTPUT VALIDATION

**Post-processing:**
1. Silent spelling correction (typos only)
2. Strip banned closers (if in thingsToAvoid)
3. Validate scene format (if BTS post)
4. Generate hashtags (disabled for most posts)

**Quality Checks:**
- Length within range
- Contains required elements (dish name for menu posts)
- No hallucinated ingredients (Free tier)
- Matches brand voice (Paid tier)

## FACTUAL DATA vs AI INVENTION

### FACTS FROM DATABASE (Both Phases)

**Completely Verified:**
- Business name (businesses.name)
- City location (business_locations.city)
- Dish names (menu_results_v2 or title)
- Dish descriptions (menu_results_v2.item_description) - **PAID ONLY**
- Opening hours (business_operations.opening_hours_json) - **PAID ONLY**
- Price level (business_operations.price_level) - **PAID ONLY**
- Weather (OpenWeather API)
- Current date/time (system)

**AI-Generated (Phase 1):**
- Post title
- Why explanation
- Photo idea
- Concrete anchor (for BTS posts)

**AI-Generated (Phase 2):**
- Caption text
- (Menu descriptions for Free tier are ABSENT, not AI-generated)

### HALLUCINATION PREVENTION

**Free Tier Strategy:**
1. **No menu data provided** to AI
2. **OBSERVER persona** forbids invention
3. **Temperature 0.0** removes randomness
4. **Explicit forbidden list** in prompt
5. **Example boundary violations** shown

**Paid Tier Strategy:**
1. **Full menu data from DB** provided
2. **Sanitized descriptions** (no upsells)
3. **Signature phrases filtered** to context
4. **Content anchors** = verified facts
5. **Brand voice rules** enforce style

### WHEN HALLUCINATIONS STILL OCCUR

**Free Tier (before OBSERVER fix):**
- ❌ Invented: "briochebolle", "chilimayo", "syltede rødløg"
- Root cause: Weak instruction "Skriv KUN fakta"
- Fix: OBSERVER persona + explicit boundaries

**Paid Tier (rare):**
- ✅ Works well (should not be touched per user request)
- Creative freedom is intentional
- Grounded in actual menu data

## SUMMARY

**Free Tier Philosophy:**
- Minimalist factual reporting
- "Describe what IS, not what could be"
- No database overhead (menu lookup skipped)
- Low AI cost (mini model, temp 0.0)

**Paid Tier Philosophy:**
- Rich storytelling with full context
- Creative expression within brand voice
- Full database access
- Premium AI quality (GPT-4o, temp 0.7)

**Key Insight:**
The system enforces tier boundaries through:
1. **Data access control** (DB queries skipped for Free)
2. **AI persona** (OBSERVER vs Brand Voice)
3. **Model selection** (mini vs full)
4. **Temperature** (0.0 vs 0.7)
5. **Prompt instructions** (prohibitions vs guidelines)
